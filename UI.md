# UI 디자인 가이드

## 1. 디자인 컨셉

- 기본 테마는 종이 노트 기반 학습 UI를 유지한다.
- 새 핵심 오브젝트는 "학습을 통해 돌보는 고양이"다.
- 감정 톤은 귀엽고 애착이 생기되, 방치 상태는 명확하게 인지되어야 한다.
- 공포나 과한 병증 표현은 피하고 모바일 캐주얼 게임 수준의 경고 표현으로 제한한다.

## 2. 핵심 화면 구조

### 2.1 온보딩

- 언어 선택
- 학습 목표 설정
- 첫 고양이 이름 짓기
- "하루 15분 공부하면 건강하게 키울 수 있어요" 안내

### 2.2 홈 대시보드

- 상단: 오늘의 학습 시간, 포인트, streak
- 중앙: 대표 고양이 카드
- 데스크톱 기준 좌측 2/3에는 대표 고양이 카드, 우측 1/3에는 `바로 학습 시작` 패널을 배치
- 대표 상태: 건강, 배고픔, 청결, 기분, 성장 단계
- 빠른 액션 버튼: 학습 시작, 먹이주기, 목욕, 놀아주기, 약먹이기
- 하단: 오늘의 학습 미션과 보상 예상 포인트

### 2.3 고양이 상세 화면

- 성장 단계별 전신 일러스트
- 현재 상태 요약 문구
- 상태 바
- 배고픔 게이지
- 청결 게이지
- 스트레스 게이지
- 건강 상태 배지
- 최근 돌봄 로그 타임라인
- 위험 상태 카운트다운
- 다음 진화 조건, 다음 보상 조건

### 2.4 학습 화면

- 학습 시작 전에 "이번 학습으로 예상 획득 포인트" 노출
- 학습 완료 후 즉시 포인트 지급 애니메이션
- 학습 결과가 고양이 상태 개선에 연결되도록 요약 패널 제공
- 예: "오늘 18분 공부로 먹이 2회, 놀이 1회 분량 확보"

### 2.5 SNS 피드 공유

- 각 피드 카드에 공유 미리보기 문구 제공
- Web Share API 우선 사용
- 미지원 환경에서는 외부 공유 URL fallback 사용
- 공유 보상 메시지와 실제 공유 성공 메시지를 분리 표시

### 2.6 멀티 캣 화면

- 잠금 슬롯 포함한 고양이 리스트
- 1년 보상으로 새끼 고양이 해금 시 신규 슬롯 강조
- 각 고양이의 상태 요약 카드 제공

### 2.7 AI 이성친구 대화 화면 (`/chat`)

#### AI 친구 프로필 영역
- 대화 영역 상단에 AI 친구 프로필 표시
- 아바타 (emoji 또는 일러스트, 학습어 국기 포함)
- AI 이름 (학습어별 고유 이름)
- "오늘 {targetLanguage}로 {name}과 대화해 보세요!" 문구
- 남은 대화 가능 시간 표시 (HH:mm)

#### 대화 버블
- 유저: 오른쪽 정렬, 파란 배경
- AI (이성친구): 왼쪽 정렬, 노란/핑크 배경
- 어드바이스 메시지: 연둣빛 진체 영역에 폰트 9번만 노출

#### 교정 패널 (우측)
- 교정 문장 표시
- "원어민이라면 이렇게 말해요" 섹션 추가
- 읽기 버튼으로 평소체/TTS 전환 (v2 확장)

#### 사용 시간 및 제한 UI
- 상단에 남은 시간 표시바 (00:30 → 00:00 카운트다운)
- 5분 단위로 로컬 보라색으로 변환
- 1분 이하일 때 빨간색으로 변환
- 0분 도달 시: 입력 영역 비활성화 + 시간 제한 모달

#### 구독 유도 모달 (비구독자케이스)
- 전체 화면을 블러마스크하는 오버레이
- 중앙 카드 콘텐츠:
  - 타이틀: "프리미엄 구독이 필요합니다"
  - 제품 간략 안내
  - "구독하러 가기" 버튼 → `/settings` 이동
  - "또는 $1로 1시간 이용하기" 버튼 → 단발성 결제 플로우
- 닫기 버튼 없음 (어떤 클릭에도 닫히지 않음, 구독상품 화면으로만 처리)

#### 시간 제한 모달 (30분 초과 또는 1시간 구매 후 시간 소진)
- 전체 화면을 블러마스크하는 오버레이
- 트에지 카드 콘텐츠:
  - 타이틀: "내일 또 만나요 💕"
  - 설명: "오늘 NN분 대화를 모두 사용했어요."
  - "1시간 연장하기" 파스텔 버튼 ($1)
  - 결제 완료 후 모달 닫힘 + 대화 재개
- 소프트 푸시업 애니메이션으로 등장

## 3. 상태 표현 규칙

### 3.1 건강 상태

- `healthy`: 밝은 표정, 안정된 포즈
- `hungry`: 밥그릇 쪽을 보는 시선, 약한 울상
- `smelly`: 작은 먼지/냄새 이펙트
- `stressed`: 귀가 약간 내려가고 표정이 예민해짐
- `sick`: 축 처진 자세, 색감 약간 저하
- `critical`: 빨간 경고 라벨과 즉시 액션 CTA
- `dead`: 직접적인 충격 연출 대신 회색 톤, 휴면 카드형 표현

### 3.2 시간 경과 경고

- 3시간 이내: 정상
- 3시간 이상 무놀이: 스트레스 주의
- 12시간 이상 무놀이: 강한 경고
- 15시간 이상 무놀이: 질병 경고 및 약/주사 CTA
- 12시간 이상 무학습: 배고픔 배지 활성화
- 24시간 이상 무목욕: 냄새 아이콘 표시
- 7일 이상 무급식: 사망 위험 최상위 경고

## 4. 인터랙션 원칙

- 홈에서는 1탭 내 돌봄 액션이 가능해야 한다.
- 위험 상태일수록 학습 CTA보다 돌봄 CTA를 먼저 보여준다.
- 돌봄에 필요한 포인트가 부족하면 부족량과 추천 학습 액션을 함께 제시한다.
- 학습 완료 직후 고양이 반응 연출을 넣어 보상 체감을 강화한다.
- 포인트 소모 확인은 과도한 모달 대신 즉시 반영형 UI를 우선한다.

## 5. 비주얼 컴포넌트 규칙

### 5.1 고양이 카드

- 일러스트 중심 카드
- 성장 단계 라벨
- 현재 상태 배지
- 남은 안전 시간 표시

### 5.2 포인트 배지

- 학습 포인트와 사용 포인트를 분리 표기
- 획득은 녹색, 소모는 주황 계열
- 잔액은 항상 홈 상단에 고정

### 5.3 상태 바

- 배고픔: 노랑에서 주황으로 변화
- 청결: 민트에서 갈색으로 변화
- 스트레스: 하늘색에서 빨강으로 변화
- 건강: 정상, 주의, 아픔, 위급 4단계 라벨

## 6. 반응형 기준

- 모바일 우선 단일 컬럼
- 태블릿 이상에서는 좌측 고양이 패널, 우측 학습/미션 패널 2열 구성
- 홈 대시보드 핵심 액션 버튼은 모바일에서 엄지 도달 범위 하단에 고정

## 7. 접근성 및 UX 원칙

- 색상만으로 상태를 구분하지 않고 텍스트 라벨과 아이콘을 함께 제공
- 상태 변화는 toast만 쓰지 말고 카드 내부에서도 확인 가능해야 함
- 죽음, 질병, 위기 상태는 사용자에게 원인과 회복 방법을 명확히 알려야 함
- 장기 미접속 사용자가 복귀했을 때 현재 상태와 필요한 최소 액션을 요약해 준다

## 8. 향후 이미지 생성 연계 가이드

- 상태별 일러스트는 동일 시드 캐릭터 느낌을 유지해야 한다.
- 프롬프트 파일명은 `성장단계-상태-버전.txt` 규칙을 사용한다.
- UI에서 실제 사용하는 상태 명칭과 프롬프트 파일명이 일치해야 한다.
- 고양이 표정, 포즈, 배경, 소품은 학습 세계관과 연결한다.

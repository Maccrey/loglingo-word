WordFlow PRD

AI 기반 글로벌 언어 학습 플랫폼에 "학습으로 돌보는 고양이 타마고치" 루프를 결합한다. 사용자는 단어 학습, 복습, 회화 학습으로 포인트를 얻고 그 포인트를 사용해 고양이를 건강하게 돌본다. 하루 15분 이상 꾸준히 공부하면 고양이 생존과 성장에 필요한 비용을 충분히 감당할 수 있도록 설계한다.

## 1. 제품 목표

### 1.1 해결하려는 문제

- 단어 암기가 실제 회화와 장기 기억으로 연결되지 않는다.
- 학습 습관이 쉽게 끊기고 복귀 동기가 약하다.
- 기존 게임화 요소는 숫자 보상에 머무르며 감정적 애착을 만들지 못한다.

### 1.2 핵심 가설

- 사용자가 학습 성과를 "내 캐릭터의 생존과 성장"으로 체감하면 D1, D7, D30 유지율이 유의미하게 오른다.
- 하루 15분 학습 보상만으로 기본 돌봄 루프를 안정적으로 유지하게 하면 과도한 페널티 없이 습관 형성이 가능하다.
- 1년 장기 육성 보상으로 추가 새끼 고양이를 지급하면 장기 리텐션과 컬렉션 욕구를 동시에 강화할 수 있다.

### 1.3 핵심 목표 지표

- 가입 전환율 60% 이상
- D1 유지율 55% 이상
- D7 유지율 40% 이상
- D30 유지율 20% 이상
- 일일 15분 학습 달성률 45% 이상
- 첫 고양이 30일 생존율 70% 이상
- 결제 전환율 5% 이상

## 2. 제품 컨셉

### 2.1 서비스 한 줄 설명

"공부할수록 건강하게 자라는 고양이를 돌보는 언어 학습 앱"

### 2.2 핵심 경험

- 학습하면 포인트를 얻는다.
- 포인트로 먹이주기, 목욕하기, 놀아주기, 약먹이기/주사맞히기를 한다.
- 돌봄 상태가 좋을수록 고양이는 새끼 고양이에서 어른 고양이로 성장한다.
- 학습을 쉬면 고양이는 배고픔, 냄새, 스트레스, 질병 상태에 들어간다.
- 장기적으로 1년 이상 안정적으로 키우면 새끼 고양이 1마리를 추가로 획득한다.

## 3. 사용자 흐름

유입 → 가입 → 언어 설정 → 첫 단어 학습 → 포인트 획득 → 첫 고양이 수령 → 홈에서 학습/돌봄 루프 반복 → 성장 단계 진화 → 장기 보상 해금

## 4. 핵심 기능

### 4.1 언어 학습

- 커리큘럼 기반 단어 학습
- 사용자 입력 단어 저장
- 틀린 단어 자동 복습 큐
- 문장 학습 및 AI 대화
- 일일 학습 시간 집계

### 4.2 고양이 타마고치 시스템

- 기본 지급 캐릭터: 새끼 고양이 1마리
- 성장 단계
- `kitten`: 시작 단계
- `junior`: 누적 육성 30일
- `adult`: 누적 육성 90일
- `middleAge`: 누적 육성 150일
- `senior`: 누적 육성 210일
- `veteran`: 누적 육성 280일
- `legacy`: 누적 육성 365일
- 365일 연속이 아니라 "1년 동안 꾸준히 건강하게 육성한 누적 기간"을 기준으로 보상 판정
- `legacy` 달성 시 새끼 고양이 1마리 추가 지급
- 추가 지급은 계정당 기본 1회, 향후 시즌 확장 가능

### 4.3 돌봄 액션

- 먹이주기: 배고픔 회복, 생존 유지
- 목욕하기: 냄새 상태 해소
- 놀아주기: 스트레스 해소
- 약먹이기: 질병 회복
- 주사맞히기: 중증 상태 회복 또는 운영 이벤트용 확장 액션

### 4.4 상태 이상 규칙

- 학습하지 않으면 12시간 후 배고픔 상태 진입
- 마지막 급식 후 7일이 지나면 사망
- 먹이를 준 뒤 24시간이 지나면 냄새 상태 진입
- 냄새 상태가 72시간 누적되면 질병 상태 진입
- 마지막 놀이 후 3시간부터 스트레스가 쌓이기 시작
- 12시간까지는 경고 상태만 표시
- 15시간이 지나면 스트레스성 질병 상태 진입
- 질병 상태에서 장시간 방치하면 사망 가능
- 약 또는 주사 사용 시 건강 상태로 복귀

### 4.5 포인트 경제

- 포인트는 학습 활동으로만 획득한다.
- 주요 적립 기준
- 일일 학습 시작
- 15분 이상 학습 완료
- 외운 단어 수
- 복습 완료 수
- 정답 streak
- 문장 학습 완료
- AI 대화 완료
- 주요 사용처
- 먹이
- 목욕
- 놀이
- 약
- 주사
- 하루 15분 이상 학습 시 기본 돌봄 비용을 감당하고도 약간의 잉여 포인트가 남는 구조를 기본값으로 설계한다.

### 4.6 운영자 조정 가능 설정

- 포인트 적립량
- 돌봄 액션별 포인트 소모량
- 배고픔/냄새/스트레스/질병/사망까지 걸리는 시간
- 성장 단계 기준 일수
- 추가 새끼 고양이 지급 조건
- 모두 `.env` 기반으로 주입하고 코드에는 안전한 기본값을 둔다.

### 4.7 AI 및 이미지 확장

- AI 대화는 OpenAI 기반으로 유지
- 향후 캐릭터 일러스트/상태 화면은 Gemini Nano Banana 등 이미지 생성 모델로 제작 가능하도록 프롬프트 자산 구조를 미리 정의한다.
- 프롬프트는 버전 관리 가능한 텍스트 파일로 보관한다.

### 4.8 현재 구현 기준 메모

- 홈 대시보드는 대표 고양이 카드와 빠른 학습 시작 패널을 한 화면에서 함께 제공한다.
- 고양이 상세 화면은 상태 게이지, 성장 기록, 현재 상태 요약 문구를 제공한다.
- 리더보드와 피드는 모두 공유 미리보기 문구를 먼저 보여주고 외부 공유로 연결된다.
- 현재 고양이 상태 저장은 프론트엔드 mock 단계로 `LocalStorage`를 사용한다.

## 5. 밸런스 원칙

- 무과금 사용자도 "매일 15분 학습"만으로 첫 고양이를 무리 없이 건강하게 유지할 수 있어야 한다.
- 미접속 패널티는 존재하되, 하루 정도의 누락으로 즉시 치명 상태가 되지 않게 완충 구간을 둔다.
- 죽음은 학습 습관 복귀를 완전히 막는 경험이 되지 않도록 부활 또는 새끼 고양이 재분양 정책을 후속 릴리즈에서 검토한다.
- 장기 보상은 희소해야 하지만 불가능해 보이면 안 된다.

## 8. 비기능 요구사항

- 학습 액션 후 포인트 반영 응답 1초 이내
- 고양이 상태 계산은 서버/클라이언트 모두 동일한 규칙을 사용
- 시간 기반 상태 전이는 재접속 시 정확히 재계산되어야 함
- 포인트 ledger는 감사 가능해야 함
- 운영자가 `.env`만 수정해 밸런스를 조정할 수 있어야 함

## 9. 리스크 및 대응

- 과도한 페널티로 이탈 가능
- 기본값을 "15분 학습이면 안전" 기준으로 설계
- 시간 계산 오류로 상태 불일치 가능
- 순수 함수 기반 상태 전이와 단위 테스트 우선
- 포인트 인플레이션 가능
- ledger와 일일 상한 정책 적용
- 이미지 생성 비용 증가 가능
- 프롬프트 자산 선정리 후 실제 생성은 후행 도입

## 10. 출시 범위

### MVP

- 언어 학습 기본 루프
- 포인트 적립/차감
- 고양이 1마리 육성
- 먹이/목욕/놀이/약
- 시간 기반 상태 이상
- 홈 대시보드와 고양이 상태 UI
- `.env` 기반 밸런스 설정

### MLP

- 주사 액션
- 다중 고양이 슬롯
- 1년 육성 보상으로 새끼 고양이 추가 지급
- SNS 공유와 고양이 성장 카드
- 이미지 프롬프트 자산 관리

### 확장

- 부활권/응급실 아이템
- 고양이 성격/희귀도
- 시즌성 스킨
- 이미지 자동 생성 파이프라인
